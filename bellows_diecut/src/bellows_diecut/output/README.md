# Bellows Diecut — generated output

Generated by `bellows_diecut.generate_tessellation(...)` — each
pattern's unit cell **tessellated** across the 170×170 print bed per the README
"Tile Specifications".  Each die is the **folded foldcore surface** (flat tilted
facets, no flat regions); male and female are the same surface offset by one
fabric thickness (matched dies).  The folded mesh is computed in NumPy
(`mesh/<name>_tile_<side>.obj`), imported into the **Tanuki DSL** and baked to STL
in **Blender**.  Regenerate with:

```python
from bellows_diecut import generate_tessellation, PATTERNS
for name in PATTERNS:   # yoshimura, miura, waterbomb, kresling, resch
    generate_tessellation(name, "output", bake=True)   # needs Blender on PATH
```

(Single unit cells — for refining a tile — come from `generate_diecut(name, …)`.)

## Contents

| Folder / file | What it is |
|---|---|
| `stl/<name>_tile_{male,female}.stl` | **Print-ready tessellated dies** (baked in Blender) |
| `mesh/<name>_tile_<side>.obj` | The computed folded foldcore mesh (imported by the DSL) |
| `obj/<name>_tile.obj` | Flat crease line mesh, grouped by fold type (GN preview) |
| `svg/<name>_tile.svg` | Flat crease reference (mountain red, valley blue dashed) |
| `json/<name>_tile.json` | Full parametric description (params + every crease) |
| `<name>_tile_diecut.py` | Self-contained Tanuki DSL → Blender bake script |
| `debug_bellows.blend` | Inspection scene: the single unit cells + coloured fold refs |

Male and female are the **same folded surface** offset by one fabric thickness
(matched dies), so pressing them forms the cloth into the folded shape. Rebuild
the unit-cell inspection scene with:

```bash
blender --background --python ../bpy_generated/build_debug.py
```
